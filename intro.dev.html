Chapter 01

npm & Package Managers

npm

Before writing a single line of TypeScript or React, you need to understand how modern JavaScript projects are structured and how dependencies are managed. npm is the cornerstone of the entire ecosystem.

What is npm?

npm (Node Package Manager) is the world's largest software registry. It ships with Node.js and serves three purposes:

  • Registry — A public database of 2M+ open-source packages at npmjs.com
  • CLI tool — A command-line tool to install, update, and manage packages
  • Dependency manager — Tracks what your project depends on and ensures reproducible installs
💡
npm vs npx vs yarn

npm installs packages. npx runs a package without installing it globally. yarn is an alternative manager by Meta with similar functionality but different internals.

package.json — The Project Manifest

Every npm project has a package.json file at its root. It describes your project's metadata and tracks all dependencies.

package.json JSON 
{
  "name": "my-app",
  "version": "1.0.0",
  "description": "My awesome app",
  "main": "index.js",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "test": "vitest",
    "lint": "eslint src --ext .ts,.tsx"
  },
  "dependencies": {
    "react": "^18.3.0",
    "react-dom": "^18.3.0",
    "axios": "^1.7.0"
  },

    "vite": "^5.2.0",

  }
}
Semantic Versioning

npm uses semantic versioning to communicate how packages change over time. Understanding this prevents accidental breaking changes.

Symbol Meaning Example Allows
^ Caret (compatible) ^18.3.0 Updates minor and patch (18.x.x)
~ Tilde (approximate) ~18.3.0 Updates patch only (18.3.x)
exact No symbol 18.3.0 Only this exact version
* Wildcard * Any version (avoid in prod!)

npm Scripts

Scripts in package.json let you define shortcuts for common tasks. Run them with npm run <name>.

Terminal

$ npm install axios              # install a package and save to dependencies
$ npm install -D typescript      # install to devDependencies
$ npm uninstall lodash           # remove a package

$ npm run dev                    # run 'dev' script
$ npm run build                  # run 'build' script
$ npx create-vite my-app         # run create-vite WITHOUT installing globally

  cd my-app && npm install && npm run dev
dependencies vs devDependencies

Which one should you use?

The distinction matters for production builds and deployment size:

  • dependencies — Code your app needs to run in production (React, axios, lodash)
  • devDependencies — Tools only needed during development (TypeScript, ESLint, testing libraries, Vite)
⚠️
Never commit node_modules

The node_modules/ folder can contain thousands of files. Always add it to your .gitignore. Teammates run npm install to regenerate it from package.json.

package-lock.json

When you run npm install, npm generates a package-lock.json file. This file:

  • Records the exact version of every dependency installed (including transitive deps)
  • Ensures every team member and CI pipeline installs the same versions
  • Should be committed to version control
  • Never manually edit this file — npm manages it automatically

Chapter 02

TypeScript Fundamentals

TypeScript

TypeScript is a typed superset of JavaScript. It compiles down to plain JavaScript but adds a powerful type system that catches bugs before your code even runs.

Why TypeScript Exists

JavaScript is dynamically typed — variable types are determined at runtime. This is flexible but leads to entire categories of bugs:

The problem with JavaScript JS 
// JavaScript — no type errors at write time
function greet(name) {
  return "Hello, " + name.toUpperCase();
}

greet("Alice");   // ✅ "Hello, ALICE"
greet(42);       // 💥 Runtime error: name.toUpperCase is not a function
greet();        // 💥 Runtime error: Cannot read property of undefined
The TypeScript solution TS 
// TypeScript — errors caught immediately in your editor
function greet(name: string): string {
  return "Hello, " + name.toUpperCase();
}

greet("Alice");   // ✅
greet(42);       // 🔴 Error: Argument of type 'number' is not assignable to parameter of type 'string'
greet();        // 🔴 Error: Expected 1 arguments, but got 0
Primitive Types

Interfaces — Typing Objects

Interfaces describe the shape of an object. They are one of TypeScript's most powerful features for structuring complex data.

interfaces.ts TypeScript 
// Defining an interface
interface User {
  id: number;
  name: string;
  email: string;
  role: "admin" | "user" | "guest";  // literal union
  createdAt?: Date;                     // optional property
}

// Extending interfaces
interface AdminUser extends User {
  permissions: string[];
  lastLogin: Date;
}

// Using the interface
const user: User = {
  id: 1,
  name: "Alice",
  email: "[email protected]",
  role: "admin"
};

// Function that accepts any User
function displayUser(user: User): string {
  return `${user.name} (${user.role})`;
}
Typed Functions
functions.ts TypeScript 
// Function with typed params and return type

  return a + b;
}

// Arrow function with explicit types

// Optional and default parameters

// Async functions

}
Generics

Generics allow you to write reusable functions and types that work with any type while maintaining type safety.

Utility Types
utility-types.ts TypeScript 
interface User {
  id: number; name: string; email: string; age: number;
}

// Partial — all properties become optional
type PartialUser = Partial<User>;
// { id?: number; name?: string; email?: string; age?: number }

// Required — all properties become required
type RequiredUser = Required<User>;

// Pick — select specific properties
type UserPreview = Pick<User, "id" | "name">;
// { id: number; name: string }

// Omit — exclude specific properties
type UserWithoutId = Omit<User, "id">;
// { name: string; email: string; age: number }

// Readonly — prevent mutation
type ReadonlyUser = Readonly<User>;

// Record — creates a dictionary type
type UserMap = Record<string, User>;

Chapter 03

React Fundamentals

React

React is a declarative UI library for building component-based user interfaces. It was created by Meta and is the most popular frontend framework in the world.

What is React?

React's core idea: your UI is a function of state. Instead of manually manipulating the DOM, you describe what the UI should look like given some data, and React handles the updates.

Virtual DOM

React uses a virtual DOM — an in-memory representation of the real DOM. When state changes, React calculates the minimum set of changes needed and applies only those to the real DOM.

React Rendering Flow
State / Props Change
↓
Re-render Component
↓
New Virtual DOM Tree
↓
Diff with Previous VDOM
↓
Patch Real DOM (minimal changes)
↓
User Sees Updated UI
JSX — JavaScript + XML

JSX is a syntax extension that lets you write HTML-like code in JavaScript. It compiles to React.createElement() calls.

State and Hooks

Hooks let you use state and other React features inside function components. useState is the most fundamental hook.

Counter.tsx TSX 
import { useState, useEffect } from "react";

function Counter() {
  // [currentValue, setterFunction] = useState(initialValue)
  const [count, setCount] = useState<number>(0);
  const [step, setStep] = useState<number>(1);

  // useEffect runs after every render (or when dependencies change)
  useEffect(() => {
    document.title = `Count: ${count}`;
  }, [count]); // dependency array — only re-run if count changes

  return (
    <div>
      <h2>Count: {count}</h2>
      <button onClick={() => setCount(c => c + step)}>+{step}</button>
      <button onClick={() => setCount(0)}>Reset</button>
      <label>
        Step: <input
          type="number"
          value={step}
          onChange={e => setStep(Number(e.target.value))}
        />
      </label>
    </div>
  );
}

src/hooks/useFetch.ts TypeScript 
import { useState, useEffect } from "react";

// Generic custom hook for data fetching
interface FetchState<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
}

export function useFetch<T>(url: string): FetchState<T> {
  const [state, setState] = useState<FetchState<T>>({
    data: null,
    loading: true,
    error: null
  });

  useEffect(() => {
    let cancelled = false;

    const fetchData = async () => {
      try {
        const res = await fetch(url);
        if (!res.ok) throw new Error(`HTTP error: ${res.status}`);
        const data: T = await res.json();
        if (!cancelled) setState({ data, loading: false, error: null });
      } catch (err) {
        if (!cancelled) setState({ data: null, loading: false, error: "Fetch failed" });
      }
    };

    fetchData();
    return () => { cancelled = true; }; // cleanup
  }, [url]);

  return state;
}
src/pages/Posts.tsx TSX 
import { useFetch } from "../hooks/useFetch";
import { Post } from "../types";

function PostCard({ post }: { post: Post }) {
  return (
    <article className="post-card">
      <h3>{post.title}</h3>
      <p>{post.body}</p>
    </article>

}

export function PostsPage() {
  const { data, loading, error } = useFetch<Post[]>(
    "https://jsonplaceholder.typicode.com/posts"
  );

  if (loading) return <p>Loading...</p>;
  if (error)   return <p>Error: {error}</p>;

  return (
    <div>
      <h1>Posts</h1>
      {data?.slice(0, 5).map(post => (
        <PostCard key={post.id} post={post} />
      ))}
    </div>
  );
}

Chapter 05

Modern Frontend Tooling

Tooling

Professional frontend development requires a solid toolchain. Each tool has a specific job, and together they create an excellent developer experience.

Vite — Next Generation Build Tool

Why Vite is so fast

Traditional bundlers like Webpack bundle your entire application before serving it. Vite leverages native browser ES Modules during development, serving files on demand. This makes the dev server start nearly instantly regardless of project size.

  • Dev server starts in milliseconds (no bundling required)
  • HMR (Hot Module Replacement) updates only changed files
  • Production builds use Rollup for optimized output
  • Plugin system compatible with Rollup plugins
vite.config.ts TypeScript 
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      "@": "/src"  // import from "@/components/Button" instead of "../../"
    }
  },
  server: {
    port: 3000,
    open: true,
    proxy: {
      "/api": "http://localhost:8080"  // proxy API calls to avoid CORS
    }
  },
  build: {
    outDir: "dist",
    sourcemap: true
  }
});

Testing with Vitest

Vitest is a Vite-native testing framework compatible with the Jest API. It's extremely fast because it shares Vite's transformation pipeline.

Button.test.tsx TSX 
import { render, screen, fireEvent } from "@testing-library/react";
import { describe, it, expect, vi } from "vitest";
import { Button } from "./Button";

describe("Button component", () => {
  it("renders with the correct label", () => {
    render(<Button label="Click me" onClick={() => {}} />);
    expect(screen.getByText("Click me")).toBeInTheDocument();
  });

  it("calls onClick when clicked", () => {
    const handleClick = vi.fn();
    render(<Button label="Save" onClick={handleClick} />);
    fireEvent.click(screen.getByText("Save"));
    expect(handleClick).toHaveBeenCalledOnce();
  });

  it("is disabled when disabled prop is true", () => {
    render(<Button label="Can't click" onClick={() => {}} disabled />);
    expect(screen.getByRole("button")).toBeDisabled();
  });
});

Exercise 3: Generic Filter Function

Write a generic TypeScript function called filterBy that takes an array of type T[] and a predicate function (item: T) => boolean, and returns the filtered array. Use it to filter a list of numbers keeping only those above 10.

filter.tsTypeScript 
function filterBy<T>(
  arr: T[],
  predicate: (item: T) => boolean
): T[] {
  return arr.filter(predicate);
}

const numbers = [5, 12, 3, 18, 7, 22];

const bigNums = filterBy(numbers, n => n > 10);
// [12, 18, 22]

// Works with strings too (generic!)
const names = ["Alice", "Bob", "Charlotte"];
const longNames = filterBy(names, n => n.length > 3);
// ["Alice", "Charlotte"]

Exercise 6: Custom useLocalStorage Hook
React + TypeScript · Intermediate

Create a custom useLocalStorage<T> hook that takes a key: string and initialValue: T, reads from localStorage on mount, and returns a [value, setValue] pair (like useState) that also persists to localStorage on change.

// Starter import { useState } from "react"; function useLocalStorage<T>(key: string, initialValue: T) { // 1. Initialize from localStorage or use initialValue // 2. Return a wrapped setter that also saves to localStorage }
✓ Solution
useLocalStorage.tsTypeScript 
import { useState } from "react";

function useLocalStorage<T>(
  key: string,
  initialValue: T
): [T, (value: T) => void] {
  // Read from localStorage or fall back to initialValue
  const [stored, setStored] = useState<T>(() => {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  // Wrapped setter that also persists
  const setValue = (value: T) => {
    try {
      setStored(value);
      localStorage.setItem(key, JSON.stringify(value));
    } catch (error) {
      console.error(error);
    }
  };

  return [stored, setValue];
}

// Usage
const [theme, setTheme] = useLocalStorage<string>("theme", "dark");

import { useState, useEffect } from "react";

interface User {
  id: number;
  name: string;
  email: string;
  phone: string;
  website: string;
}

function UserProfile() {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users/1")
      .then(res => {
        if (!res.ok) throw new Error("Network error");
        return res.json() as Promise<User>;
      })
      .then(data => { setUser(data); setLoading(false); })

  if (error)   return <div>Error: {error}</div>;
  if (!user)    return null;

  return (
    <div>
      <h2>{user.name}</h2>
      <p>{user.email}</p>
      <p>{user.phone}</p>
    </div>
  );

✓ Solution

import { useState } from "react";

interface SearchableListProps {
  items: string[];
}

function SearchableList({ items }: SearchableListProps) {
  const [query, setQuery] = useState("");

  // Derive filtered list from state — no extra useState needed!
  const filtered = items.filter(item =>
    item.toLowerCase().includes(query.toLowerCase())
  );

  return (
    <div>
      <input
        type="text"
        value={query}
        onChange={e => setQuery(e.target.value)}
        placeholder="Search..."
      />
      <p>{filtered.length} results</p>
      <ul>
        {filtered.map((item, i) => (
          <li key={i}>{item}</li>
        ))}
      </ul>
    </div>
  );
}
